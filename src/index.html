What's TailwindCSS?

  1. Utility-first的CSS框架。
  2. 有很多預先寫好的utility class所組成。
  3. 由很多的單一特性組合次成一個元件。

Why TailwindCSS?

  1. 不用再想class名稱、數字使用上能更有系統。
  2. Utility class 能重複被用，CSS檔案也會Purge。
  3. 低階 -> 高度客製。有先蓋好的utitlity，但要重蓋也沒問題。
  4. Tailwind Intellisense提示讓學習門檻變低。
  5. 可以自訂專屬於專案的class，也能透過即時編譯。

Tailwind的CSS語感

1. 數值是[數字]*4px，由小到大間距為1、2、4、8、16

h-0(0px) h-px(1px) h-0.5(2px) h-1(4px) h-1.5(6px) h-4(16px) h-64(256px) h-72(288px) h-80(320px) h-96(384px)

2. 上(t)、下(b)、左(l)、右(r)、橫向(x)、縱向(y) + padding(p), margin(m), space-[x/y]-[number], gap-[x/y]-[number] 

          
mx-10  // margin-left: 40px; mragin-right: 40px;
my-5   // margin-top: 20px; margin-bottom: 20px;
pt-2   // padding-top: 8px;
pr-4   // padding-right: 16px;
pb-6   // padding-bottom: 24px;
pl-8   // padding-left: 32px;
p-px   // padding: 1px;
